¿Cómo optimizar consultas N+1 en Django ORM?

Tengo un modelo de Productos con relación a Categorías y al listar 100 productos hace 101 consultas SQL. ¿Cómo puedo optimizar esto?

class Product(models.Model):
    name = models.CharField(max_length=100)
    category = models.ForeignKey(Category, on_delete=models.CASCADE)

# Esto genera N+1 queries
products = Product.objects.all()[:100]
for p in products:
    print(p.category.name)  # Query extra por cada uno
